# 06. Màn Hình Admin

## Admin Dashboard

Mục đích: tổng quan quản trị.

Chức năng cần có:

- Sidebar admin.
- Header admin.
- Thống kê topic, từ vựng, bài test, user.
- Link nhanh đến quản lý topic, từ vựng, đề, user, chấm bài.
- Khi bấm menu sidebar chỉ reload phần body/workspace giữa trang, không reload toàn trang.

## Admin Quản Lý Topic

Chức năng cần có:

- Danh sách topic.
- Thêm topic.
- Sửa topic.
- Xóa topic.
- Form nhập title, slug, mô tả, part, difficulty, sample questions.

## Admin Quản Lý Từ Vựng

Chức năng cần có:

- Danh sách từ vựng.
- Tìm kiếm/lọc.
- Thêm từ.
- Sửa từ.
- Xóa từ.
- Form word, nghĩa, ví dụ, topic, level, collocation.

## Admin Quản Lý Đề Luyện

Chức năng cần có:

- Danh sách đề Reading/Listening/Writing/Speaking.
- Tạo đề mới.
- Sửa đề.
- Xóa đề.
- Upload audio nếu Listening.
- Thêm câu hỏi.
- Thêm đáp án.
- Thêm giải thích.
- Cấu hình thời gian làm bài.
- Bật/tắt xuất bản.

## Admin Chấm Bài Học Viên

Chức năng cần có:

- Danh sách bài Writing/Speaking đã nộp.
- Lọc chờ chấm/đã chấm.
- Mở bài làm.
- Nhập điểm.
- Nhập feedback.
- Lưu phản hồi để user xem.

## Admin Quản Lý User

Chức năng cần có:

- Danh sách user.
- Tìm user.
- Xem email/ngày tạo.
- Cấp/gỡ quyền admin.
- Xem trạng thái học nếu cần.

Yêu cầu giao diện admin:

- Không dùng header/footer của user.
- Admin hiển thị full layout riêng.
- Có sidebar và header rõ ràng.
- Dễ dùng, không rối.
