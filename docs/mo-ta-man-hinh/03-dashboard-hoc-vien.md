# 03. Dashboard Học Viên

## Dashboard Sau Khi Đăng Nhập

Mục đích: là giao diện chính của user đã đăng nhập, dùng để quản lý quá trình học.

Chức năng cần có:

- Sidebar điều hướng học viên.
- Header hoặc khu vực chào user.
- Tổng quan tiến độ học.
- Mục tiêu học tập.
- Bài đã làm gần đây.
- Lỗi sai cần ôn.
- Từ vựng cần ôn.
- Feedback Writing/Speaking từ admin.
- Gợi ý bước học tiếp theo.

Yêu cầu:

- Khi user đã đăng nhập, không show Home public.
- User vào `/` sẽ tự chuyển sang dashboard.
- Giao diện khác rõ với giao diện khách.

## Trang Quản Lý Tài Khoản

Mục đích: user chỉnh thông tin cá nhân.

Chức năng cần có:

- Xem tên.
- Xem email.
- Sửa tên.
- Đổi mật khẩu nếu có.
- Xem mục tiêu học.
- Link quay lại dashboard.

## Trang Lịch Sử Bài Làm

Mục đích: user xem lại các bài đã làm và lỗi sai.

Chức năng cần có:

- Danh sách bài đã làm.
- Loại bài.
- Điểm số.
- Thời gian làm bài.
- Ngày làm bài.
- Đáp án đúng/sai.
- Giải thích lỗi sai.
- Feedback nếu là Writing/Speaking.
- Trạng thái chờ admin chấm.

Yêu cầu:

- Khách chưa đăng nhập không lưu lịch sử.
- Chỉ user đăng nhập mới xem được lịch sử bài làm.
